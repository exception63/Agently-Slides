是给 AYI 时代重做的 slides。

PPT 把内容锁进二进制；AI 改不动、版本控不了、改一句要等渲染。我们换条路：让 HTML 本身就是真相。

01
WHY

先想清楚
谁该做什么

工具的形状，由"人和 AI 怎么分工"决定。

一句话理念

人做高频细活，AI 做模糊重活。

点字、换色、加动画——你即时手动，零等待零 token；"这页太满，重排成三栏"——交给 AI。

分工 · 各干各擅长的

两只手，各管一摊

人 · 即时手动
高频细活
  • 点文字直接改
  • 换配色 / 字号 / 对齐
  • 加入场动画 / 持续动效
  • 移动、删除、加减页
AI · 经一句评论
模糊重活
  • "这页太满，精简成 3 个要点"
  • "把这段做成时间轴"
  • "换更稳重的配色"
  • 只改你点名的那页

HTML 自有 · 单文件 · 可移植

成品就是一个 .html，双击即放映、能 git diff、拷到任何电脑都能跑 —— 不绑任何软件、不连任何云。

02
HOW IT'S WIRED

三块，
怎么搭起来

浏览器里改 · 本地桥接 · Claude Code 回写。

数据怎么流

一条本地闭环

前台
浏览器 Studio
你看到的编辑器：点字、换色、留评论、💾 保存 HTML。
中间
本地 Bridge
本机中间人（HTTP + WebSocket + MCP）。不出本机、不上云。
后台
Claude Code
经 MCP 读你的评论、按 data-id 精准回写那一页。
关键设计：Bridge 只在 localhost 转发，deck 状态在内存；你点"保存 HTML"才落盘成单文件。

完整流程

四步，从内容到现场

STEP 1
出片
说"做slides"，选风格 → editorial-slides 出一份单文件 deck。
STEP 2
协作精修
Studio 里自己改细节；复杂的留评论发给 Claude 回写。
STEP 3
配讲稿
transcripts_html 配同视觉讲稿，锚点对齐。
STEP 4
接演讲
presenter-mode：翻页时副屏 / 弹窗自动跟随讲稿。
两种触发方式

说人话，或者打斜杠

↳ 自动触发 slidesmith:editorial-slides（自然语言即可）
你：/slidesmith:transcripts_html
