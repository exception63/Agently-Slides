## v0.2 · 用户反馈驱动迭代（2026-05-26 下午）

### 改进 1 · 真正 1:1 映射

**做法**：

1. 在讲稿 HTML 同一小节内插入细粒度锚点（h4 / blockquote / golden / pre 上加 id）
2. 用 Python 批量脚本（`anchor-insertion-script.py.template`）一次插 73 个新锚点
3. SLIDE_MAP / SLIDE_MAP_LOCAL 改为 145 个独立锚点
4. 副屏 applyState 用 `s.anchor !== lastAnchor` 比较代替 `s.slideIdx !== lastIdx`

**关键决策**：锚点放**已有结构元素**上（h4 / blockquote / golden / pre / interact / table）· 不插 span 锚。理由：
- 减少 HTML 增量
- 已有元素本身就是视觉锚（更直观）
- 不破坏现有 DOM 结构

### 改进 2 · 整块高亮

**做法**：

1. 新增 CSS `.presenter-block-current` 类 · 用 `box-shadow: -8px 0 0 -2px gold` 画外左金条（不占布局 + 不和已有 border-left 打架）+ 浅金 bg
2. 为 h4 / pre.prompt / table 各做适配变体（保留它们的特定样式）
3. JS 在 `scrollToAnchor` 末尾调 `highlightBlock(el)` · 沿 nextElementSibling 链向后走 · 直到下一个 sN 锚点为止 · 给每个元素加 class
4. 维护 `currentBlockEls` 数组 · 翻页时一次性清掉旧高亮

## 关键决策（v0.1 → v0.2 演化）

| 决策点 | v0.1 | v0.2 | 学到了什么 |
|---|---|---|---|
| 映射粒度 | 粗（71 锚点 · 多 slide 共享） | 1:1（145 锚点 · 全独立） | 粗映射是 anti-pattern · 必须 1:1 |
| 锚点插入方式 | 手 Edit 现有 h3.sub | Python 脚本批量插 73 个新锚 | 70+ 锚点手插不可靠 |
| 锚点宿主元素 | 仅 h3.sub | h4 / blockquote / golden / pre / interact / table | 已有结构元素是天然锚 · 比插 span 好 |
| 视觉反馈 | 仅 h3.sub 红边脉冲 | h3.sub 红边 + 整块金色高亮 | 用户需要"整段"反馈 · 不只"小节" |
| 重跳逻辑 | slideIdx 比较 | anchor 比较 | 鲁棒：偶发同锚不重跳 · 保住手动滚位 |

## v0.1 → v0.2 学到的事 · 反哺到 SKILL.md

v0.2 把以下教训写入了 SKILL.md / templates / references：

1. **红线 ⭐⭐⭐**：1:1 映射是底线 · 不允许多 slide 共享锚点

7. **JS 工程**：沿 `nextElementSibling` 链遍历 · 用 id 模式匹配（`/^s\d/`）+ class 检查（`appendix`）+ tag 检查（`h3.sub`）三重停止条件

## v0.3 · 双向联动 + 提词 + 拆按钮（2026-05-27 现场前一日）

### 触发用户反馈（3 件一次性提）

1. **"键盘和翻页笔在副屏失灵"** — 焦点在演讲者模式时按 ←/→ 不翻页，因为 iframe 吞键盘
2. **"讲稿挨着念效果不好 · 关键词高亮（但不突兀）会更好"** — 现场紧张时需要提词锚点
3. **"点了讲稿目录 · slides 没联动 · 还得手动翻回去"** — 操作流断裂

### 关键设计决策

**1. "板块"颗粒度** — 讲稿 anchor 一共 150 个 · 其中 145 个对应 slide（5 个是附录 sN-data）

- 第一轮误判：把"板块"理解为 h3 子标题颗粒（82 个）→ 用户纠正"每一个 slide 就是一个板块"
- 修正后：用 sN-x 锚点 ↔ slide 1:1 映射作为"板块"边界 · 复用 v0.2 已有的 `.presenter-block-current` 类
- **教训**：拿到模糊需求要先看代码已有结构 · 别自己脑补颗粒度

**2. 按钮拆分语义** — 一个"自动跟随"按钮承担两件事 → 概念混乱

- v0.2：「🔓 自动跟随」一个开关 + 「↻ 同步」一个一次性按钮 = 5 按钮 + 1 旋转
- v0.3：「→ 跟随 / ← 跟随」两个 toggle + 「→ 同步 / ← 同步」两个一次性 = 4 + 缩放组 + Keywords + 旋转 = 8 按钮（4 行 grid）
- 红绿色态：跟随类用 `state-on: #6FAA82` 玉绿 / `state-off: #C76B5B` 赤陶 · 一眼看出当前开关

**3. 防回声** — 双向广播的核心难题

```js
// slides 端 broadcast 加 source: 'slides'
// presenter 端 reverse-broadcast 加 source: 'presenter'
// 收方守门
applyState(s)              { if (s.source === 'presenter') return; ... }
handleReverseJump(payload) { if (payload.source !== 'presenter') return; ... }
```

**4. iframe 键盘转发**

```js
// 讲稿端检测：仅在被嵌入演讲者模式时启用
if (!window.parent || window.parent === window) return;
// keydown listener → postMessage 'fuquan-nav-key' 给 parent
// parent 端 handleKey(key) 抽出统一入口 · keydown + message 都调
```

**5. Keywords 高亮的"不突兀"设计**

```css
body.cue-on :is(.presenter-block-current, .presenter-current) strong {
  color: var(--vermilion);
  /* 底部 55%-92% vermilion-tint 半透明渐变 · 水彩刷感 */
  background: linear-gradient(180deg,
    transparent 0%, transparent 55%,
    rgba(245, 227, 228, 0.85) 55%,
    rgba(245, 227, 228, 0.85) 92%,
    transparent 92%);
}
```

- 现状讲稿里有 1525 个 `<strong>` · 直接复用作关键词源 · 不打新标
- 关键：用 `:is(.presenter-block-current, .presenter-current) strong` 限定范围 · 只当前激活板块亮
- 字号 / 字重不变 · 不破坏排版节奏 · 0.25s 平滑过渡不闪烁
