# Slidesmith 使用指南（人类向 · v2）

一个 **AI 生成 + 人类可视化精修** 的 HTML 演示系统。
新版思路(v2)很简单:**AI 直接做出一份 HTML slides → 你把它拖进 Studio 改 → 导出**。

---

## 0. 三个核心概念（30 秒）

- **HTML deck** = 你这份演示的本体,一个**单文件 `.html`**(离线、可移植、U 盘拷走就能放)。
  AI 按一套"契约结构"生成它;它**本身就是真相源**——你改的、导出的,都是它。
- **Studio** = `studio/slidesmith-studio.html`,一个**双击就能打开**的网页工作台(像 Keynote)。
  **这是你的主要工具**:把 HTML deck 拖进去,就能可视化地改、然后导出。
- **Submit to AI** = 遇到复杂改动(换布局、重排、加图表)时,把**那一页**连同你的要求导出成一个小文件,
  交给 AI 改完再放回来——只改那一页,别页不动。

> 主流程:**让 AI 做一份 HTML slides → 拖进 Studio 改字/换色/调动效 → 复杂的用 Submit to AI → 导出 HTML / PDF**。

---

## A. 让 AI 帮你做 slides

对一个接入了本项目的 AI(如 Claude Code),说人话即可:

> "用 Slidesmith 帮我做一份 8 页的项目汇报,暗色 keynote 风,内容是 ……(给它你的要点 / 文档 / 大纲)。"

AI 会读根目录的 **`AGENTS.md`** 和 **`docs/DECK-CONTRACT.md`**,按契约生成一份**单文件 HTML deck**
(每页结构规整、配色用统一令牌、自带翻页/段导航/缩略图)。它还会自检一下排版有没有溢出/看不清。

拿到之后:
- 想直接放映:**双击那个 `.html`** 就能全屏放。
- 想再改:把它**拖进 Studio**(见下)。

## B. 你自己用 —— 主要靠 Studio（零门槛）

### 打开
双击 **`studio/slidesmith-studio.html`**。浏览器里就开了,不用装东西、不用联网、不用终端。

### 你能做的事
1. **导入**:把 AI 给你的 `.html` deck 拖进窗口(或点右上角"导入 HTML")。
   - 导入后它会原样显示——**连它自带的段导航 + 缩略图都在**;Studio 自己的左侧页列表会自动收起(左上角 **☰** 可随时展开/收起)。
2. **改文字**:在中间预览里**直接点标题/正文/要点**就能改。
3. **换样子**(右侧检查器):
   - **配色**:改强调色 / 背景色 / 文字色(整套即时变);主题(暗/亮/对比)用 deck 自带顶栏的切换钮;
   - **选中元素**:点中预览里某个元素 → 调它的字号 / 颜色 / 粗细 / 对齐 / 入场动画。
4. **视觉自检**:点「🔍 检查这套 deck」,自动列出哪几页有**溢出 / 对比度太低 / 坏图**,点条目跳过去修。
5. **Submit to AI**(复杂改动):见下面 C 节。
6. **导出**(右上角):
   - **导出 HTML** — 存成编辑后的单文件 deck(可再拖回来接着改);
   - **导出 PDF** — 打开打印窗口,在对话框里选「另存为 PDF」(一页一张)。

> ⚠️ Studio 不会自动保存。**改完记得「导出 HTML」**,否则关掉窗口就没了。

### 放映
双击导出的 `.html`:`←``→` 翻页 · `F` 全屏 · deck 自带的段导航/缩略图点一下就跳。

## C. Submit to AI 怎么用（以及它到底怎么工作的）

**怎么用:**
1. 点选要改的那一页(或那页里某个元素)→ 右侧「目标页」会显示是哪一页;
2. 在框里**用大白话写要怎么改**(例:"改成左右两栏,左边放定义、右边放一张趋势示意图,整体更克制");
3. 点 **① 导出修改请求** → 下载一个文件 `<deck>.<页id>.ai-request.md`;
4. 把这个文件**交给 AI**(在 Claude Code 里说"读取这个文件,按 AGENTS.md 改这一页");
5. AI 返回一小段 HTML(就那一页),你点 **② 应用 AI 返回** 选那个文件 → 那一页就换好了。

**常见疑问(重要):**

- **我写的内容怎么传给 AI?** 走文件、手动交接(Studio 离线、不联网、不内嵌 AI)。
  那个请求文件是**自包含**的——里面有你的指令、这一页当前的完整 HTML、配色令牌——所以 AI 哪怕全新会话也能照着改。
- **AI 怎么只改我要的那页?** 请求文件里**只有那一页**,AI 看不到别页;返回时也被要求"只给那一页、保留同一个 `data-id`"。
  Studio 收到后**按这个 id 只替换那一页**,其余每页原封不动。
- **人和 AI 同时改会冲突吗?** 它是**轮流**的,不是同时,正常不冲突:导出请求时会带上你最新的改动;应用补丁时会先保住你**别页**的改动,再只换目标页。
  **唯一要注意**:一页"正发给 AI 改"的时候,别再自己手动改**那一页**(否则 AI 基于旧快照改完,会盖掉你那页的新手改)。别的页随便改。

## D. 命令行（可选,给愿意用终端的人 / 自动化）

```bash
npm install                                  # 仅第一次
npm run build:studio                         # 改了 Studio 源码后重新打包
# 可选的"数据格式(IR)"管线(强约束 / 批量 / 无头导出 PDF·PNG):
npm run sm -- new mydeck                      # 生成模板
npm run sm -- build mydeck.deck.md -o dist    # 出 HTML deck
npm run sm -- doctor mydeck.deck.md           # 校验 + 质检 + 视觉自检(出问题截图)
npm run sm -- export mydeck.deck.md -o dist -f pdf
```

> 数据格式(JSON IR)是一条**可选的、更受约束的**老路;v2 主路是"AI 直接生成 HTML deck"。详见 `AGENTS.md`。

## E. 什么时候用本项目

| 你的情况 | 更适合 |
|---|---|
| 一次性、追求单页视觉惊艳、用完即弃 | 直接让 AI 自由生成 HTML(更快) |
| 要反复改、自己不调 AI 也能改字换色、要离线投屏 + PDF、要复杂改动能回灌 AI | **Slidesmith** |

### 现在还不擅长的（诚实告知）
- **自由拖拽 / 像素级摆位**还没有(刻意缓做——它和"可重排、可换肤"的底子冲突)。
- **导入 deck 的双屏演讲者 / 讲稿同步**还没接(deck 自带的演讲者钮另说);PDF 已有。
- 视觉自检覆盖溢出 / 对比度 / 坏图;不替你做审美判断。

## F. 常见问题
- **双击 Studio 空白?** 用 Chrome / Edge / Safari 打开;离线 HTML,不需联网。
- **导入后没反应?** 确认是符合契约的 HTML deck(每页 `<section class="slide">`);AI 按 `DECK-CONTRACT.md` 生成的就符合。
- **中文字体不好看?** 用的是系统字体栈(离线优先);装了思源 / 苹方会更好。
- **想把 Studio 发给别人?** 直接发 `slidesmith-studio.html` 这**一个文件**,双击即用。
- **改了没保存就关了?** Studio 不自动保存——务必先「导出 HTML」。
