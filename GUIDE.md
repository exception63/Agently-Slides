# Slidesmith 使用指南（人类向）

一个**浏览器里改 HTML slides、Claude Code 实时帮你改**的编辑器。
核心是**分工**:你做高频细活(点字、换色、加动画),AI 做模糊重活(你留一句**评论**,它改完回写)。

---

## 0. 三个核心概念（30 秒）

- **HTML deck** = 你这份演示的本体,一个**单文件 `.html`**(离线、可移植)。它**本身就是真相源**。
- **Studio** = 那个网页工作台(像 Keynote)。你在这里可视化地改、给 AI 留评论、看进度、导出。
- **评论(任务)** = 复杂/模糊的改动,你在**那一页**上用大白话写一句(如"改成左右两栏、右栏放个关键数字"),
  攒几页一起 **🚀 发送给 Claude**;它只改你点名的页,改完那页打 **✓ 已改**,不满意能 **↩︎ 还原**。

> 主流程:**连上 Claude → 你点字换色调动效(自己动手)+ 给难改的页留评论发给 Claude → 看着它一页页变 ✓ → 导出**。

## 0.5 连接 Claude（你问得最多的）

**桥接是连接点。** 只要 **Claude Code 开着**(装了 `slidesmith` 插件),桥接就一直在
`http://localhost:8765` 上跑(插件的 MCP 服务在 Claude Code 启动时就把它拉起来了)。

**方向一 · 从 Claude 发起(最常用)**
跟 Claude 说"用 slidesmith 打开 这个 deck",或敲 `/slidesmith <deck.html>`。
Claude 调 `slidesmith_open` → 你的默认浏览器**自动弹出 Studio,已经连上**(顶栏绿 **● 已连接 Claude**)。

**方向二 · 先开 Slidesmith,再连 Claude**
关键:**从 `http://localhost:8765` 打开**(浏览器输这个地址),不是双击文件。
只要 Claude Code 开着,这地址就是活的,打开即自动连上。
然后让 Claude 载入 deck(`slidesmith_open`),或直接把 deck 文件拖进 Studio。
> 如果你**双击了文件**(`file://…slidesmith-studio.html`)→ 那是**离线模式**,不会连 Claude。要连就改用 localhost:8765。
> 如果 Claude Code **没开** → 桥接不在,跑 `npm run sm -- serve <deck>` 自己把它拉起来,或干脆开 Claude Code。

**连接状态长什么样**
- **已连接**:顶栏一颗绿 **● 已连接 Claude**;发送按钮是「🚀 发送给 Claude」。
- **离线/未连**:没有那颗绿胶囊;发送按钮是「📦 导出 N 个任务」(走手动文件)。
- 让 Claude 跑 `slidesmith_status` 也能查:几个 Studio 连着、当前哪份 deck、几个任务待处理。

**`/clear` 之后怎么续上**
`/clear` 只清对话、**不关 Claude Code**,所以桥接还在 8765 上跑。
直接再 `/slidesmith <deck>`(或让 Claude 打开)即可;你那个浏览器 tab 通常还连着。

## A. 让 AI 帮你做 slides

对一个接入了本项目的 AI(如 Claude Code),说人话即可:

> "用 Slidesmith 帮我做一份 8 页的项目汇报,暗色 keynote 风,内容是 ……(给它你的要点 / 文档 / 大纲)。"

AI 会读根目录的 **`AGENTS.md`** 和 **`docs/DECK-CONTRACT.md`**,按契约生成一份**单文件 HTML deck**
(每页结构规整、配色用统一令牌、自带翻页/段导航/缩略图)。它还会自检一下排版有没有溢出/看不清。

拿到之后:
- 想直接放映:**双击那个 `.html`** 就能全屏放。
- 想再改:把它**拖进 Studio**(见下)。

## B. 你自己用 —— 主要靠 Studio（零门槛）

### 打开
双击 **`studio/slidesmith-studio.html`**。浏览器里就开了,不用装东西、不用联网、不用终端。

### 你能做的事
1. **导入**:把 AI 给你的 `.html` deck 拖进窗口(或点右上角"导入 HTML")。
   - 导入后它会原样显示——**连它自带的段导航 + 缩略图都在**;Studio 自己的左侧页列表会自动收起(左上角 **☰** 可随时展开/收起)。
2. **改文字**:在中间预览里**直接点标题/正文/要点**就能改。
3. **换样子**(右侧检查器):
   - **配色**:改强调色 / 背景色 / 文字色(整套即时变);主题用 deck 自带顶栏的切换钮;
   - **选中元素**:点中预览里某个元素 → 调它的字号 / 颜色 / 粗细 / 对齐 / 入场动画 / 持续动效,
     还能 **↑上移 / ↓下移 / 🗑删除** 这个元素。这些**高频细活你自己动手最快**(即时、不耗 AI)。
4. **视觉自检**:点「🔍 检查这套 deck」,自动列出哪几页有**溢出 / 对比度太低 / 坏图**,点条目跳过去修。
5. **让 AI 改**(复杂/模糊的改动):见下面 C 节——这是和 Claude 协作的主路。
6. **导出**(右上角):
   - **导出 HTML** — 存成编辑后的单文件 deck(可再拖回来接着改);
   - **导出 PDF** — 打开打印窗口,在对话框里选「另存为 PDF」(一页一张)。

> ⚠️ Studio 不会自动保存。**改完记得「导出 HTML」**,否则关掉窗口就没了。

### 放映
双击导出的 `.html`:`←``→` 翻页 · `F` 全屏 · deck 自带的段导航/缩略图点一下就跳。

## C. 让 AI 改（评论工作流，主路）

**怎么用(连上 Claude 时):**
1. 翻到要改的那一页 → 右侧「✨ 让 AI 改」会显示**本页是第几页**(这个框**永远跟着你正在看的那页**走);
2. 用大白话写一句**评论**(例:"改成左右两栏,左边放定义、右边放趋势示意图,整体更克制");写完自动记住;
3. 可以**切到别的页继续写**——左侧每页会冒出 ● 徽标、右侧「全部任务」队列在涨;
4. 还能在「**对整份 deck 说…**」写一句全局的(如"统一所有页标题字号"),AI 会自己挑相关页改;
5. 攒好了点 **🚀 发送给 Claude** → Claude 自动接到、按契约改、回写;
6. 不用招呼它:它改完哪页,那页当场更新、徽标变 **✓ 已改**;不满意点 **↩︎ 还原本页**回到改之前。

**进度怎么看(发完不用猜):**
- **● 橙** = 待发送(写了还没发) · **● 蓝(脉冲跳动)** = 已发送、Claude 正在改 · **✓ 绿** = 已改。
- 发送后顶部还会亮一条蓝色横幅「**● 已发送 N 个任务,Claude 正在改…**」,改完一页数字减一,到 0 消失。

**常见疑问(重要):**
- **AI 怎么只改我要的那页?** 每页有个稳定 `data-id`,Claude 返回时按 id **只替换那一页**,其余每页原封不动。
- **人和 AI 会冲突吗?** 是**轮流**不是同时:发送时带上你最新的改动,回写时先保住你**别页**的改动再只换目标页。
  唯一注意:一页**正发出去等 Claude 改**的时候(蓝色脉冲),别再自己手改**那一页**。别的页随便改。
- **离线时呢?** 没连 Claude 也能用:写好评论点「📦 导出 N 个任务」下个文件,交给任意 AI;它返回的 `<section data-id>` 你点「应用粘贴 / 从文件应用」放回去。

> ⚠️ **Studio 不自动存盘**,桥接里的 deck 也只在内存。改告一段落**务必「导出 HTML」**存下来(或让 Claude 帮你存)。

## D. 命令行（可选,给愿意用终端的人 / 自动化）

```bash
npm install                                  # 仅第一次
npm run build:studio                         # 改了 Studio 源码后重新打包
# 可选的"数据格式(IR)"管线(强约束 / 批量 / 无头导出 PDF·PNG):
npm run sm -- new mydeck                      # 生成模板
npm run sm -- build mydeck.deck.md -o dist    # 出 HTML deck
npm run sm -- doctor mydeck.deck.md           # 校验 + 质检 + 视觉自检(出问题截图)
npm run sm -- export mydeck.deck.md -o dist -f pdf
```

> 数据格式(JSON IR)是一条**可选的、更受约束的**老路;v2 主路是"AI 直接生成 HTML deck"。详见 `AGENTS.md`。

## E. 什么时候用本项目

| 你的情况 | 更适合 |
|---|---|
| 一次性、追求单页视觉惊艳、用完即弃 | 直接让 AI 自由生成 HTML(更快) |
| 要反复改、想自己点字换色 + 难改的让 Claude 实时帮改、要离线投屏 + PDF | **Slidesmith** |

### 现在还不擅长的（诚实告知）
- **自由拖拽 / 像素级摆位**还没有(元素只能 ↑↓ 排序、删除,刻意缓做)。
- **整份 deck 的字体/配色令牌**改不了走 AI 回写(只换得了页内 `<section>`);这类用右侧检查器自己改。
- **导入 deck 的双屏演讲者 / 讲稿同步**还没接;PDF 已有。
- 视觉自检覆盖溢出 / 对比度 / 坏图;不替你做审美判断。

## F. 常见问题
- **顶栏没有绿色「已连接」?** 你多半是双击文件打开的(离线模式)。要连 Claude,改从 `http://localhost:8765` 打开(需 Claude Code 开着,或先 `serve`)。见 0.5 节。
- **点了发送没反应/不知道发出去没?** 看徽标:变**蓝色脉冲**=发出去了、Claude 正在改;顶部也有蓝色横幅。改完自动变 ✓。Claude 是连上后自动接活的。
- **改完想让 Claude 重新连/换一份 deck?** 让它跑 `slidesmith_open <别的deck.html>`,会重新载入并连上。
- **双击 Studio 空白?** 用 Chrome / Edge / Safari;离线 HTML,不需联网。
- **导入后没反应?** 确认是符合契约的 HTML deck(每页 `<section class="slide">`)。
- **想把 Studio 发给别人?** 直接发 `slidesmith-studio.html` 这**一个文件**,双击即用(离线模式)。
- **改了没保存就关了?** Studio 不自动保存——务必先「导出 HTML」。
